Testbed: Jest Basics

Experimentell

  • Installation 1: npm i -D @types/jest jest jest-environment-jsdom
  • Installation 2: "karma" durch "jest" in angular.json ersetzen: `@angular-devkit/build-angular:jest`
  • Installation 3: "jasmine" durch "jest" in "tsconfig.spec.json" ersetzen und "test.ts" aus include Array entfernen.
  • Installation 4: Karma und Jasmine deinstallieren: npm uninstall @types/jasmine jasmine-core karma karma-chrome-launcher karma-coverage karma-jasmine karma-jasmine-html-reporter
  • Installation 5: Unnötige Dateien entfernen: `rm ./karma.conf.js ./src/test.ts`

Alternativ: Angular Builders verwenden

  • Link: https://github.com/just-jeb/angular-builders/tree/master/packages/jest
  • Installation 1: s. o. Installation 3 + 4
  • Installation 2: `npm i -D jest @types/jest @angular-builders/jest`
  • Installation 3: s. o. Installation 3
  • Installation 4: Builder "karma" durch "@angular-builders/jest:run" in angular.json ersetzen: `@angular-builders/jest:run`

Im Anschluss funktioniert Jest größtenteils wie Karma, nur das Jest deutlich schneller ist, da es Tests parallel ausführen kann und kein echten Webbrowser benötigt.

Auf der anderen Seite unterstützt Jest nicht alle nativen Browser Funktionen/APIs, wie ResizeObserver, LocalStorage usw.
