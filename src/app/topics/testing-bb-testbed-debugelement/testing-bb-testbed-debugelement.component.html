DOM-Manipulationen und Auslösen von Events, wie "click" usw.

  • Host: Mit Host ist immer die Component gemeint, die gerade getestet wird bzw. in der Component Klasse selbst das Element im DOM, welches von Angular eingefügt wird.
  • classes: CSS Klassen des "Host"
  • attributes: HTML Attribute des Host (data- oder aria-Attribute uvm.)
  • triggerEventHandler: Löst Event-Handler von Angular aus. Entweder von bestimmten Elementen (die mit query gesucht wurden) oder auf das Host Element.
  • query: Kann Elemente im Template der Component suchen. Dafür braucht es die Hilfsklasse `By.css('... css selector ...')`
  • nativeElement: Zugriff auf das echte DOM Element. Hierauf können auch native Events aufgerufen werden (u. a. `click()`), was sich aber schlechter mocken lässt.
